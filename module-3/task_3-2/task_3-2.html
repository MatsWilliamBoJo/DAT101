1. Can Part 1 be solved with a single "for" loop?

    Yes — use two index variables in the for header (for example: for (let i=1, j=10; ...)) and build both lines at once.

  2. Can we create an "infinite" loop inside the while condition?

    No — the while condition must be boolean. Use while(true) or a function that always returns true and break from the loop when done.

  3. How fast or slow is the guessing loop and what happens while it runs?

    Expected attempts for random guessing ≈ N for range size N. Small N is fast; large N can be slow. A synchronous loop blocks the JS main thread (UI freezes) unless you use Web Workers or async techniques.

  4. How do rows and columns relate in nested for loops? Can we change the order?

    Typically outer loop = rows, inner = columns, producing row-by-row output. Swapping them changes traversal (column-first) and the output order; both are valid depending on desired layout.
